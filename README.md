# Field-Advisor (Flask + ML)

Field-Advisor is a Flask web application that helps farmers make better decisions using machine learning:

- **Crop recommendation** based on soil nutrients and weather
- **Fertilizer suggestion** based on nutrient gaps for a selected crop
- **Plant disease detection** from a leaf image

The UI is served with Flask templates and the ML inference runs locally using pre-trained models.

---

### Features

- **Crop recommendation** (scikit-learn): predicts the best crop using:
  - Soil: **N, P, K, pH, rainfall**
  - Weather: **temperature + humidity** (fetched from OpenWeatherMap by city)
- **Fertilizer suggestion**: compares your N/P/K vs recommended values in `app/Data/fertilizer.csv`
- **Plant disease detection** (PyTorch): classifies an uploaded image using a ResNet9 model
- **HTML + JSON responses**: endpoints return JSON when called via AJAX (`X-Requested-With: XMLHttpRequest`)

### Quickstart (Windows)

From the repository root:

1) Create & activate a virtual environment

```bash
python -m venv .venv
.venv\Scripts\activate
```

2) Install dependencies

```bash
pip install -r requirements.txt
```

If PyTorch install fails on your machine, install it separately (CPU example):

```bash
pip install torch torchvision torchaudio --index-url https://download.pytorch.org/whl/cpu
```

3) Add your OpenWeatherMap key

Create `app/.env`:

```bash
weather_api_key=YOUR_OPENWEATHERMAP_KEY
```

Notes:
- The project reads either `weather_api_key` or `WEATHER_API_KEY`.
- `.env` files are already ignored by `.gitignore` (do not commit secrets).

4) Add required models

Create the folder `app/models/` and place:

- `RandomForest.pkl` (crop recommendation)
- `plant_disease_model.pth` (disease detection)

5) Run the Flask app (important: run from `app/` so relative paths work)

```bash
cd app
python app.py
```

Then open `http://127.0.0.1:5000/`.

### Web routes (what the app exposes)

- **Home**: `GET /`
- **Crop**
  - `GET /crop-recommend` (form)
  - `POST /crop-predict` (prediction)
- **Fertilizer**
  - `GET /fertilizer` (form)
  - `POST /fertilizer-predict` (recommendation)
- **Disease**
  - `GET /disease-predict` (upload page)
  - `POST /disease-predict` (image inference)

---

### Model inputs

#### Crop recommendation

The model is called with:

- **Soil (user input)**: `N`, `P`, `K`, `pH`, `rainfall`
- **Weather (API)**: `temperature`, `humidity` from OpenWeatherMap by `city`

Final feature vector:

\[
[N, P, K, temperature, humidity, ph, rainfall]
\]

#### Fertilizer suggestion

Uses `app/Data/fertilizer.csv` to look up recommended N/P/K per crop and returns guidance for the most deviating nutrient.

#### Disease detection

Uploads an image (`file`) and runs a PyTorch model (`ResNet9`) to predict one of the PlantVillage-style classes.

### Training / generating models

Notebooks live in `notebooks/` and cover data preparation + model training.

- **Crop model**: `notebooks/Crop_Recommendation_Model.ipynb`  
  Train multiple classifiers and export the best model as a pickle (example used by the app: `RandomForest.pkl`).

After training, ensure the file ends up at:

- `app/models/RandomForest.pkl`

If the notebook saves to a different folder (e.g. `models/` at repo root), copy it into `app/models/`.

The plant disease `.pth` file is **not generated by a notebook in this repo** (as currently provided). Add your trained weights as:

- `app/models/plant_disease_model.pth`

### Troubleshooting

- **App crashes on startup with “No such file or directory: models/…”**
  - You are missing model files or running from the wrong directory.
  - Run from `app/` and ensure `app/models/RandomForest.pkl` and `app/models/plant_disease_model.pth` exist.
- **Weather fetch fails**
  - Confirm your API key is valid and rate limits aren’t hit.
  - Try `WEATHER_API_KEY` (uppercase) or `weather_api_key` in `app/.env`.
- **PyTorch install issues on Windows**
  - Install torch separately using the official PyTorch index URL (see Quickstart).

---

### Security note

Never commit API keys. Keep them in `app/.env` locally or set environment variables in your deployment platform.
